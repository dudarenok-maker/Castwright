## Benefit / Rationale

- **User:** n/a ‚Äî no user-visible change.
- **Technical:** every actionable dependency is current across the three npm/pub
  surfaces (root + server npm, Flutter pub); the three majors each carry a
  deliberate behavior/floor note (express-rate-limit IPv6 `/56` masking, sharp
  libvips/engines floor, connectivity_plus Android toolchain floor) instead of
  surprising a future `npm install`. Keeps the audit signal clean so the next
  round starts from zero.

## Architectural impact

- **No new seams.** Lockfile + manifest version bumps only; no application source changed.
- **Invariants preserved:** Node `>=20.19.0` floor; the two-lockfile boundary
  (server deps bumped via `cd server`); server `@types/node` stays `^20`
  (isolated from root `^25`); eslint stays `^9` (`ops-14` upstream-blocked); the
  Python sidecar engine deps are untouched (deferred to `side-17`).
- **Migration story:** drift 2.34 regenerated `library_database.g.dart`
  (byte-identical ‚Äî `schemaVersion => 5` ladder unchanged); express-rate-limit 8
  adds an `ip-address` transitive and IPv6 `/56` grouping (benign for the
  single-user 1000/min limiter; the `standardHeaders: true` header surface did
  not flip ‚Äî the locking test passed unchanged); sharp 0.35 refreshed the
  per-platform `@img/sharp-*` matrix (verified intact, no triple pruned).
- **Reversibility:** each major is its own revertable commit; `git revert` one
  and re-run `verify` to drop it without unwinding the round.
