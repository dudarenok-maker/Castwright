> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Drive every actionable dependency current across the three live npm/pub surfaces (frontend npm, server npm, Flutter pub), correct the three upstream-blocked backlog notes, and file the deferred Python-sidecar engine-dep spike — all in one PR.

**Architecture:** One branch (`chore/deps-round-4`, already cut) with three scoped commit groups — `chore(deps)` for JS (root + server), `chore(app)` for Flutter, `docs(deps)` for the plan doc + backlog reconciliation. The repo is a **two-lockfile npm workspace with no `workspaces` field**: root `package.json`/`package-lock.json` is one surface; `server/package.json`/`server/package-lock.json` is a second, reached only via `cd server` (a root `npm update` never touches it). Flutter is the third surface (`apps/android/`). The Python sidecar is a fourth surface, audited but **deferred** (its engine deps are safety-pinned and need a GPU box + golden-audio gate).

**Tech Stack:** npm (Node ≥20.19), Vite/React/TS frontend, Node/Express server (sharp, express-rate-limit), Flutter/Dart companion (drift codegen, connectivity_plus), husky verify gates.

**Source spec:** `docs/superpowers/specs/2026-06-18-deps-round-4-design.md` (thrice adversarially reviewed). Read it for the full audit and the per-major migration notes.

- **Node floor `>=20.19.0`** (root `package.json:7`). All bumps must clear it (they do).

- **Do NOT touch** `server/package.json`'s `@types/node` (`^20`, deliberately matches the Node-20 runtime — isolated from root's `^25`).
- **Do NOT touch** the Python sidecar (`server/tts-sidecar/requirements/*.txt`) — deferred this round.

- **Keep `tailwindcss` + `@tailwindcss/postcss` in lockstep** (both → 4.3.1).
