# QA-gate short-sentence false positives — PR-1 (gate-logic fixes) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Eliminate the short-sentence QA false positives that drive bimodal Qwen-1.7B per-chapter RTF, with zero loss of real-defect detection.

**Architecture:** Three pure-function gate-logic fixes in the existing QA modules — (A1) an absolute-length floor on the duration "runaway" branch, (A2a) fuzzy compound-bridging in the ASR gate, (A2b) a short-reference single-substitution evidence backstop — each behind a registry knob. No sidecar, no I/O: every change is unit-testable in `vitest` against in-memory PCM / injected transcripts. The fixes remove the wasted single-synth re-records (RTF ~6) that dilute the healthy batched RTF (~0.7); they do **not** disable any gate.

**Tech Stack:** TypeScript, Node, Vitest. Server-side only (`server/src/tts/`, `server/src/config/`).

## Global Constraints

- **Branch:** all PR-1 work lands on `fix/server-qa-gate-false-positives` (already cut off `main`; the design spec commit `5dfaf40b` is on it).
- **Commit convention:** `<type>(<scope>): <subject>`. Scope for every commit here is `server`. `fix(server): …` for the gate fixes; `test(server): …` is also valid. **`tts` is NOT an allowed scope** — it was rejected once already.
- **Source of truth for spec:** `docs/superpowers/specs/2026-06-30-qa-gate-false-positives-and-rtf-telemetry-design.md` (§ PR-1).
- **Registry knobs default to current behaviour at their floor.** Every new knob ships with a default that the existing tests already exercise; `minRunawaySec=3.0`, `minRefWords=2`. Setting a knob to `0` disables the new guard (restores pre-PR behaviour) — keep that property.
- **No `--no-verify`.** Run `cd server && npm run test` (or the targeted file) before each commit; the pre-commit hook is scope-filtered and will run the server leg.
- **TDD, every task:** failing test → run-red → minimal impl → run-green → commit.
- **Real-defect invariant (do not regress):** a dropped word / deletion run, a repetition (high `compressionRatio`), a multi-error garble, and a genuine (≥3 s) runaway must all still flag after every task.

## Decisions resolved from the spec's open questions

- **OQ1 (A1):** floor **alone**, no fixed-overhead term added to `expectedSec` (an overhead term feeds the *same* expression both duration branches divide by → would false-trip the truncation branch on fast short lines).
- **OQ2 + OQ3 (A2):** ship the **`qa.asr.minRefWords` evidence backstop** (single *substitution* on a ≤N-word reference → `inconclusive`, deletions/insertions exempt) **instead of** phonetic/metaphone name-matching. Rationale: lighter (no new dependency — OQ3's stated preference), more general (covers every short-reference homophone, not just cast names: `nodded`→`Notted`, `Valkyrie`→`Volkery`), and the deletion-exempt clause preserves negation-flip detection. Metaphone is **not** implemented in PR-1.
- **OQ4 (3 near-silent pads):** out of PR-1 scope — not touched here.

```typescript
/** True when `a` and `b` are within Levenshtein distance 1 (equal, one
    substitution, or one insertion/deletion). Bounded short-circuit — no full DP
    matrix — so it is cheap to call per token pair. Used by bridgeCompounds to
    tolerate the one-character drift Whisper introduces when it re-segments a
    compound ("skulduggery" → "skull duggery" → "skullduggery", one extra l). */
export function editDistanceAtMost1(a: string, b: string): boolean {
  if (a === b) return true;
  const la = a.length;
  const lb = b.length;
  if (Math.abs(la - lb) > 1) return false;
  let i = 0;
  while (i < la && i < lb && a[i] === b[i]) i += 1;
  if (la === lb) return a.slice(i + 1) === b.slice(i + 1); // one substitution
  if (la > lb) return a.slice(i + 1) === b.slice(i); // one deletion from a
  return a.slice(i) === b.slice(i + 1); // one insertion into a
}
```

Replace the body of `bridgeCompounds` (the `collapse` closure + the `expSet`/`actSet` lines) with:

```typescript
export function bridgeCompounds(expected: string[], actual: string[]): [string[], string[]] {
  // Collapse an adjacent pair when its concatenation matches a token in the
  // OTHER stream exactly OR within edit-distance 1, and emit that matched token
  // so the two streams align as a `match` rather than a residual substitution.
  // A genuinely wrong pair won't concatenate near an other-stream token, so this
  // can never mask real drift. Pairs only (2↔1); 3+ token compounds out of scope.
  const collapse = (tokens: string[], other: readonly string[]): string[] => {
    const out: string[] = [];
    for (let i = 0; i < tokens.length; i += 1) {
      if (i + 1 < tokens.length) {
        const concat = tokens[i] + tokens[i + 1];
        const match = other.find((o) => editDistanceAtMost1(concat, o));
        if (match !== undefined) {
          out.push(match);
          i += 1; // consumed the pair
          continue;
        }
      }
      out.push(tokens[i]);
    }
    return out;
  };
  return [collapse(expected, actual), collapse(actual, expected)];
}
```

```typescript
/* One-off, read-only measurement (PR-1 validation). Re-derives the ASR + duration
   QA verdicts over a rendered book's *.segments.json with the CURRENT code, so we
   can quote before/after false-positive counts in the spec Ship notes. Not wired
   into any harness — run on the box that has the corpus:
     npx tsx server/scripts/qa-gate-dryrun.ts "<audio dir with *.segments.json>"
   No writes. Resolves the expected-text/transcript/signals straight from each
   segment record (schema: { text, qa:{...}, asr:{ transcript, ... } }). */
import { readdirSync, readFileSync } from 'node:fs';
import { join } from 'node:path';
import { classifyTranscript, type AsrSignals } from '../src/tts/segment-asr-qa.js';
import { DEFAULT_SEGMENT_QA_THRESHOLDS } from '../src/tts/segment-qa.js';

const dir = process.argv[2];
if (!dir) {
  console.error('usage: qa-gate-dryrun.ts <audio-dir>');
  process.exit(1);
}

let total = 0;
let asrDriftNow = 0;
let durLongNow = 0;
const t = DEFAULT_SEGMENT_QA_THRESHOLDS;

for (const f of readdirSync(dir).filter((n) => n.endsWith('.segments.json'))) {
  const segs = JSON.parse(readFileSync(join(dir, f), 'utf8')) as any[];
  for (const s of segs) {
    total += 1;
    // A1 re-evaluation from recorded duration vs expected.
    const dur = s.qa?.durationSec;
    const exp = s.qa?.expectedSec;
    if (dur != null && exp != null && exp > 0) {
      const ratio = dur / exp;
      if (ratio > t.maxDurationRatio && dur >= t.minRunawaySec) durLongNow += 1;
    }
    // A2 re-evaluation from recorded transcript.
    const text: string | undefined = s.text;
    const transcript: string | undefined = s.asr?.transcript;
    if (text && transcript != null) {
      const signals: AsrSignals = {
        avgLogprob: s.asr?.avgLogprob ?? null,
        noSpeechProb: s.asr?.noSpeechProb ?? null,
        compressionRatio: s.asr?.compressionRatio ?? null,
      };
      const c = classifyTranscript(text, transcript, signals);
      if (c.verdict === 'drift') asrDriftNow += 1;
    }
  }
}

console.log(`segments scanned: ${total}`);
console.log(`ASR drift (post-PR-1 code): ${asrDriftNow}`);
console.log(`duration "runaway" (post-PR-1 code): ${durLongNow}`);
console.log('Compare against the spec baseline: 121 drift + 51 long (pre-PR-1).');
```
